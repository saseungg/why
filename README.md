#  why 

> 제가 생각하는 좋은 개발자란 항상 '왜'라는 질문을 고민하며 기술에 접근하는 사람입니다.
답을 정확히 내리지 못할지라도 어떤 기술을 사용하거나 학습할 때, 그 기술의 필요성이나 동작 방식을 깊게 고민하는 것과 그렇지 않은 것 사이에는 큰 차이가 있다고 생각합니다.
제가 추구하는 개발자로 성장하기 위해 학습하면서 '왜'라는 질문을 내린 것들을 기록하는 공간입니다.
